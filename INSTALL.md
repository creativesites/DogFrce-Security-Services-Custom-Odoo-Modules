# Installation and Setup Guide

This guide walks through a full local setup of the DogForce Security Suite: Odoo backend, PostgreSQL database, demo data, and the optional mobile app.

For architecture and module design, see [ARCHITECTURE.md](ARCHITECTURE.md).

---

## Prerequisites

### Required (Odoo backend)

| Tool | Minimum version | Notes |
|------|-----------------|-------|
| **Docker Desktop** | 4.x+ | Includes Docker Compose v2. [Download](https://www.docker.com/products/docker-desktop/) |
| **Git** | 2.x+ | To clone the repository |
| **Bash-compatible shell** | — | macOS, Linux, or WSL2 on Windows |

Verify Docker is installed and running:

```bash
docker --version          # Docker 24+ recommended
docker compose version    # Compose v2.x
```

The Odoo backend runs entirely inside Docker. You do **not** need Python or PostgreSQL installed on the host for normal development.

### Optional (mobile app)

| Tool | Minimum version | Notes |
|------|-----------------|-------|
| **Node.js** | 18.x LTS or 20.x LTS | Required for Expo / React Native |
| **npm** | 9.x+ | Bundled with Node |
| **Expo Go** (physical device) or **Xcode / Android Studio** (simulator) | — | For running the mobile app |

Verify Node (only if working on `mobile/`):

```bash
node --version   # v18.x or v20.x
npm --version    # 9.x+
```

### Optional (local Python tooling)

| Tool | Version | When you need it |
|------|---------|------------------|
| **Python** | 3.10+ (Odoo 19) | IDE linting, scripts outside Docker, contributing to Odoo core |

Custom module code is executed inside the Odoo container (Python 3.12 on the official `odoo:19.0` image).

### Platform images (pinned by this repo)

| Service | Image | Defined in |
|---------|-------|------------|
| Odoo | `odoo:19.0` | `deploy/docker-compose.yml` |
| PostgreSQL | `postgres:16` | `deploy/docker-compose.yml` |

## 1. Clone the repository

```bash
git clone <repository-url>
cd "DogFrce Security Services Custom Odoo Modules"
```

---

## 2. Environment variables

### Create `.env`

Copy the example file and edit values if needed:

```bash
cp .env.example .env
```

`./scripts/start.sh` will also create `.env` from `.env.example` automatically if it is missing.

### Variable reference

| Variable | Default | Description |
|----------|---------|-------------|
| `COMPOSE_PROJECT_NAME` | `dogforce-odoo-dev` | Prefix for Docker container names |
| `ODOO_VERSION` | `19.0` | Odoo Docker image tag |
| `POSTGRES_DB` | `postgres` | PostgreSQL default database name |
| `POSTGRES_USER` | `odoo` | PostgreSQL username |
| `POSTGRES_PASSWORD` | `odoo` | PostgreSQL password |
| `ODOO_DB` | `dogforce_dev` | Odoo application database name |
| `ODOO_ADMIN_PASSWORD` | `admin` | Odoo master password (web database manager) |
| `ODOO_PORT` | `8069` | Host port for Odoo HTTP |
| `ODOO_LONGPOLLING_PORT` | `8072` | Host port for Odoo longpolling / bus |

Example `.env.example` (also at the repo root):

```env
COMPOSE_PROJECT_NAME=dogforce-odoo-dev
ODOO_VERSION=19.0
POSTGRES_DB=postgres
POSTGRES_USER=odoo
POSTGRES_PASSWORD=odoo
ODOO_DB=dogforce_dev
ODOO_ADMIN_PASSWORD=admin
ODOO_PORT=8069
ODOO_LONGPOLLING_PORT=8072
```

### Generated files (do not commit)

| Path | Purpose |
|------|---------|
| `.env` | Your local environment (gitignored) |
| `.local/odoo.conf` | Odoo config generated by `start.sh` |
| `.local/postgres/` | PostgreSQL data volume |
| `.local/odoo/` | Odoo filestore and session data |

## 3. Start the Odoo stack

From the repository root:

```bash
chmod +x scripts/*.sh    # first time only, if scripts are not executable
./scripts/start.sh
```

This script:

1. Loads `.env`
2. Creates `.local/` directories
3. Writes `.local/odoo.conf` from environment variables
4. Starts PostgreSQL and Odoo via Docker Compose

Wait until Odoo is healthy, then open:

```text
http://localhost:8069
```

## 4. Create and seed the database

You can seed the database from the command line (recommended) or through the Odoo web UI.

### Option A — Automated seed (recommended)

Install the Security Suite modules and load Namibian demo data in one step:

```bash
./scripts/seed-db.sh
```

This runs inside the Odoo container:

```bash
odoo -c /etc/odoo/odoo.conf -d dogforce_dev \
  -i security_demo_data,security_mobile,security_documents,security_accounting_controls,security_reporting \
  --stop-after-init
```

Installing `security_demo_data` pulls in the full core dependency chain:

`security_base` → `security_operations` → `security_attendance` → `security_leave` → `security_l10n_na` → `security_payroll_core` → `security_loans` → `security_discipline` → `security_billing` → `security_client_reports` → `security_equipment` → `security_fleet`

The post-install hook in `security_demo_data` then creates:

- Namibia payroll rules and tax brackets
- Grades, certifications, clients, sites, and posts
- Demo guards with bank and tax details
- Roster slots and attendance records
- Loans, incidents, equipment, fleet, payroll, and billing samples

First run typically takes **3–10 minutes** depending on hardware.

### Option B — Manual seed via web UI

1. Open **http://localhost:8069**
2. Create a database:
   - **Master Password:** value of `ODOO_ADMIN_PASSWORD` (default `admin`)
   - **Database Name:** value of `ODOO_DB` (default `dogforce_dev`)
   - **Email / Password:** your admin login (default Odoo suggests `admin` / `admin`)
   - **Language:** English
   - **Country:** Namibia (loads NAD currency)
   - **Demo data:** optional — the Security Suite has its own demo module
3. Enable **Developer Mode** (Settings → Activate Developer Mode)
4. Go to **Apps**, remove the "Apps" filter, and install:
   - `Security Company Namibia Demo Data` (`security_demo_data`)
   - Optionally: `Security Mobile API`, `Security Documents`, `Security Accounting Controls`, `Security Reporting`

### Log in after seeding

| Field | Default |
|-------|---------|
| URL | `http://localhost:8069` |
| Database | `dogforce_dev` (or your `ODOO_DB` value) |
| Login | `admin` |
| Password | `admin` (unless you chose a different password in the web UI) |

## 5. Install mobile app dependencies (optional)

Only required if you are running the DogForce Mobile app locally.

```bash
cd mobile
npm install
```

Configure the Odoo server URL if not using localhost:

```typescript
// mobile/src/api/client.ts
export const ODOO_BASE_URL = 'http://localhost:8069';
```

For a physical device, use your machine's LAN IP (e.g. `http://192.168.1.10:8069`) instead of `localhost`.

Start the Expo dev server:

```bash
npm start
```

Then press `i` (iOS simulator), `a` (Android emulator), or scan the QR code with Expo Go.

The mobile app requires the `security_mobile` module to be installed on Odoo. Mobile users must be linked to `hr.employee` records and assigned the appropriate security groups (Supervisor, Manager, or Owner) in Odoo.

## 6. Verify the installation

### Odoo backend

1. Log in at `http://localhost:8069`
2. Open **Security** (or related app menus) and confirm demo records exist:
   - Guards under **Employees**
   - Client sites and roster batches under **Security Operations**
   - Posting sheets under **Security Attendance**
   - Payslips under **Security Payroll**

### Run module tests

See [TESTING.md](TESTING.md) for the full guide. Quick start:

```bash
./scripts/run-tests.sh security_payroll_core
```

Other modules with tests: `security_equipment`, `security_fleet`.

### Mobile API smoke test

With Odoo running and `security_mobile` installed, authenticate and call an endpoint (replace session as needed):

```bash
curl -s -X POST http://localhost:8069/web/session/authenticate \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"call","params":{"db":"dogforce_dev","login":"admin","password":"admin"}}'
```

## Updating modules after code changes

After pulling new code or editing Python/XML in `custom_addons/`:

1. Restart Odoo (or upgrade the changed module):

```bash
./scripts/stop.sh && ./scripts/start.sh
```

2. Upgrade specific modules from the container:

```bash
docker compose -f deploy/docker-compose.yml exec odoo odoo \
  -c /etc/odoo/odoo.conf \
  -d dogforce_dev \
  -u security_payroll_core \
  --stop-after-init
```

Or use **Apps → Upgrade** in the Odoo UI (Developer Mode required).

## Resetting the local database

To start completely fresh:

```bash
./scripts/stop.sh
rm -rf .local/postgres .local/odoo
./scripts/start.sh
./scripts/seed-db.sh
```

This destroys all local Odoo and PostgreSQL data. Back up first if needed:

```bash
./scripts/backup-db.sh
```

## Troubleshooting

### Docker is not on PATH (macOS)

Ensure Docker Desktop is running. The helper script checks common paths:

```bash
/Applications/Docker.app/Contents/Resources/bin/docker
```

### Port 8069 already in use

Change `ODOO_PORT` in `.env` (e.g. `8070`), then restart:

```bash
./scripts/stop.sh && ./scripts/start.sh
```

### Module not found in Apps list

1. Confirm `custom_addons/` is mounted: check `deploy/docker-compose.yml`
2. Update the Apps list: **Apps → Update Apps List** (Developer Mode)
3. Remove the "Apps" search filter

### `seed-db.sh` fails with database connection error

Wait for PostgreSQL to become healthy after `./scripts/start.sh`:

```bash
./scripts/logs.sh
```

Look for `database system is ready to accept connections` before re-running seed.

### Mobile app cannot reach Odoo

- Use your host machine IP, not `localhost`, when testing on a physical device
- Ensure Odoo port is exposed and not blocked by a firewall
- Confirm the database name in the login screen matches `ODOO_DB`
